문제 / 배경
배포된 앱은 언제든 공격 대상이 됩니다

앱스토어에 올라간 순간 코드는 누구나 내려받아 분석할 수 있습니다. 세 가지 공격 시나리오가 지금 이 순간에도 진행 중입니다.

THREAT 01
역공학으로 앱 코드가 그대로 노출되고 있지 않습니까?
JADX·APKTool 같은 공개 툴로 누구나 소스코드를 복원할 수 있습니다. 비즈니스 로직, API 키, 인증 알고리즘이 그대로 드러납니다.
THREAT 02
루팅·탈옥 기기에서 결제와 인증이 우회되고 있지 않습니까?
루팅·탈옥 기기는 앱의 보안 체계를 OS 수준에서 무력화합니다. 금융·게임 서비스의 핵심 피해가 여기서 발생합니다.
THREAT 03
앱이 변조·재배포되어 악성 버전이 유통되고 있지 않습니까?
정상 앱에 악성코드를 삽입해 재배포하는 리패키징 공격은 브랜드 신뢰도와 사용자 피해를 동시에 유발합니다.

FAQ
자주 묻는 질문

라이선스, 지원 환경, 인증 등 도입 전 궁금한 사항을 정리했습니다.

라이선스 방식은 어떻게 되나요?
+
영구형과 구독형 중 선택 가능합니다. 영구형은 초기 구매 후 유지보수 계약을 통해 업데이트를 받으며, 구독형은 연간 계약으로 업데이트와 기술지원이 포함됩니다. 고객사 환경에 맞는 방식을 권장해드립니다.
지원하는 OS 버전은 어디까지인가요?
+
Android 5.0(Lollipop)부터 최신 Android 15까지, iOS는 iOS 13 이상을 지원합니다. 신규 OS 출시 시 사전 베타 테스트를 통해 즉각 대응합니다.
React Native, Flutter 등 크로스플랫폼 프레임워크도 지원하나요?
+
React Native, Flutter, Ionic 모두 지원합니다. 크로스플랫폼 환경에서도 네이티브와 동일한 수준의 보안을 적용할 수 있습니다.
기존 CI/CD 파이프라인과 연동이 가능한가요?
+
네, 기존 빌드 파이프라인을 변경하지 않고 연동 가능합니다. On-Premise, AWS, Cloud 등 모든 인프라 환경을 지원하며, APK/IPA 업로드·다운로드 과정이 자동화됩니다.
GS 인증 외에 다른 수상 실적이 있나요?
+
GS(Good Software) 인증 1등급 외에 과학기술정보통신부 주관 대한민국 SW제품 품질대상 최우수상, SK텔레콤 모바일기술대상을 수상했습니다.
금융보안원 가이드라인 준수 여부를 어떻게 확인하나요?
+
금융보안원 전자금융기반시설 보안 취약점 분석·평가 방법 안내서의 전 항목에 대한 공식 준수 문서를 제공합니다. 도입 검토 시 자료 요청을 통해 수령하실 수 있습니다.
보안 기능 적용 후 앱 성능에 영향을 주나요?
+
성능 영향을 최소화하도록 설계되어 있습니다. 실제 고객사 적용 결과 앱 실행 속도 및 배터리 소모에 유의미한 영향이 없었음을 확인했습니다. 도입 전 POC를 통해 직접 확인하실 수 있습니다.
